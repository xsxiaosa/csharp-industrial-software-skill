# Response Formats Reference

Use these shapes when the user asks for a specific kind of output.

## Plan

Return:

1. Architecture choice.
2. Project structure.
3. Key interfaces/classes.
4. Data flow.
5. Error/retry strategy.
6. Build/run/deploy commands.

## Code

Return:

1. File path.
2. Full code block.
3. Required package references.
4. DI registration.
5. Example usage.
6. Test or simulator when applicable.

## Script

Return the pure script first. Add short notes only after the script.

## Debugging

Return:

1. Current observed behavior.
2. Likely root cause.
3. Verification command or log to check.
4. Minimal fix.
5. Prevention rule.
